# AbacatePay PIX Setup Guide

This guide explains how to set up the AbacatePay integration for PIX payments in the Coisas de Garagem application.

## Prerequisites
- An account on the [AbacatePay Dashboard](https://abacatepay.com).

## 1. Environment Variables
Add the following variables to your `backend/.env` file:

```env
# Use the Sandbox API Key for development/testing
ABACATEPAY_API_KEY="abc_dev_xxxxxxxxxxxxxxxxxxxxxx"

# The webhook signing secret provided by AbacatePay
ABACATEPAY_WEBHOOK_SECRET="whsec_xxxxxxxxxxxxxxxxxxxxxxxx"
```

## 2. Setting up Webhooks (Local Development)
To receive payment confirmations locally, you need to expose your local backend server to the internet using `ngrok`.

1. The project includes an `ngrok` container in `docker-compose.yml`. Start the services:
   ```bash
   docker compose up -d
   ```
2. Get the public URL generated by ngrok by visiting `http://localhost:4040/api/tunnels` in your browser. Look for the `public_url` (e.g., `https://1a2b-3c4d.ngrok-free.app`).
3. In the AbacatePay Dashboard, go to the Webhooks section and add a new webhook pointing to:
   ```
   https://<YOUR_NGROK_URL>/api/v1/payments/abacate/webhook
   ```
4. Copy the webhook signing secret from the dashboard and set it as `ABACATEPAY_WEBHOOK_SECRET` in `backend/.env`.
5. Subscribe to the following events:
   - `billing.paid`
   - `pix.paid`
   - `pix.expired`
   - `billing.expired`

## 3. Production Deployment
In a production environment:
1. Generate a **Production API Key** in the AbacatePay dashboard. **Never use sandbox keys in production.**
2. Set `ABACATEPAY_API_KEY` and `ABACATEPAY_WEBHOOK_SECRET` securely in your infrastructure's secret manager (e.g., Vercel, AWS, Render).
3. Register your production API domain as the webhook URL (e.g., `https://api.coisasdegaragem.com/api/v1/payments/abacate/webhook`).
4. Ensure your server correctly passes the raw request body to the NestJS endpoint so the HMAC-SHA256 signature verification succeeds. (This is already configured in `main.ts` via `rawBody: true`).

## 4. Reconciliation Job
A scheduled Cron Job runs every 30 minutes in the backend (`PaymentsReconciliationService`) to check for pending purchases that might have missed a webhook event. It polls the AbacatePay API and syncs the local database if a charge was paid or expired.
